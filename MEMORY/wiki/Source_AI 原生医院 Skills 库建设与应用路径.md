# Source: AI 原生医院 Skills 库建设与应用路径

## 核心内容概览
本文定义了 AI 原生医院的核心交付单元——**Skills 库**。它将传统的臃肿业务逻辑拆解为可被 AI 调用的原子化能力单元。

## 关键技术架构
- **EAC 模型 (Entity-Action-Constraint)**：Skill 能力单元的物理定义标准。Entity 定义操作对象，Action 定义动作类型，Constraint 定义医学红线与合规约束[^1]。
- **[[Entity_ACE.md]] (Agent Coordination Engine)**：负责在复杂临床路径中实时调度并编排这些原子 Skills[^1]。
- **能力热力图**：通过可视化手段识别医院临床能力的数字化覆盖度与逻辑盲区[^1]。

## 核心论点
- Skills 库是医院实现“数字资产入表”的物理载体，也是训练专科 Agent 的核心语料[^1]。
- 未来的医院信息化不是购买软件，而是订阅不断进化的能力单元（Skill Store）[^1]。
