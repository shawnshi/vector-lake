# 第二十七讲：项目管理与交付：确保蓝图不只是“墙上的画”

## 核心概述
本讲聚焦于医疗数字化项目的落地“最后一公里”，将项目管理定义为“在约束中跳舞的艺术”。深入剖析了医疗 IT 项目失败的核心原因——范围蠕变、沟通断层与价值缺位，并提出了从“功能交付”向“价值交付”转期的实战方法论[^1]。

## 关键知识点

### 1. 医疗 IT 项目的“铁三角”约束
- **范围 (Scope)**：最易失控的变量。必须防范 [属于:: [[Concept_Scope_Creep.md]]][^1]。
- **时间 (Time)**：受制于评审节点与院长任期。
- **成本 (Cost)**：软件商的利润红线与医院的预算硬伤。

### 2. 核心风险控制：CCB 机制
- **[[Concept_CCB_变更控制委员会.md]]**：建立刚性的变更管理流程，杜绝工程师私下接受主任的非标需求[^1]。

### 3. 交付价值论
- **蓝图与现实的断裂**：项目延期或失败往往是因为售前承诺的“星辰大海”在交付阶段退化为“一地鸡毛”[^1]。
- [Relation:: [[Concept_Value_Driven_Delivery.md]]]

## 关键论点
- **项目经理是变革者**：他管理的不只是代码，更是各方利益的博弈与预期[^1]。
- **验收不是终点，是价值实现的起点**[^1]。
