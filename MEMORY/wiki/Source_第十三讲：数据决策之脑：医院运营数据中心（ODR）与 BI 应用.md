# 第十三讲：数据决策之脑：医院运营数据中心（ODR）与 BI 应用

## 核心摘要
本讲定义了 **ODR (Operation Data Repository)** 为医院的“决策之脑”，并强调其与传统数据仓库的区别：ODR 的核心使命是“回答问题”。通过 BI 工具与 ODR 的结合，医院管理层可以获得“上帝之眼”，实现从经验驱动向数据驱动的权力转移。

## 关键观点
- **从“数据坟场”到“决策之脑”**: 医院建设 ODR 的唯一目的是支撑决策。若数据无法转化为行动，则只是沉淀的负资产。
- **数据治理的“市政工程”**: ODR 的构建涉及主数据管理 (MDM)、ETL 及主题库建设，是长周期的底层基建 [支持:: [[Concept_数据治理.md]]]。
- **权力转移与组织摩擦**: 数据透明化会触动既有利益格局（如科室主任的自主权），ODR 的推行不仅是技术问题，更是组织变革问题。

## 提取的实体与概念
- **[[Entity_运营数据中心_ODR.md]]**: 医院运营管理的数据中枢。
- **[[Concept_数据治理.md]]**: ODR 建设的前提与保障。
- **[[Entity_BI系统.md]]**: 数据的可视化呈现与交互入口。

## 总结
ODR + BI 构成了医院智慧管理的闭环，通过将海量碎片数据转化为战略洞察，为医院在变局中提供确定性的决策依据。
