# 网络价值乘数 (Network Value Multiplier)

## 定义
网络价值乘数是衡量 AI 应用场景价值的高阶维度，用于评估单点问题的解决对整个外部价值网络（如医保支付、分级诊疗、公共卫生监控等）的影响深度和撬动效应[^1]。

## 核心内涵
- **外部性量化**：不同于传统的内部效率评估，乘数强调的是场景与外部节点的连接性。例如，一个能精准预防医保扣费的 AI 场景，其网络价值远高于一个单纯提高排班效率的场景[^1]。
- **共振效应**：寻找内部痛点（如医生写病历慢）与外部压力（如医保局对病历质量的刚性要求）产生共振的交汇点[^1]。
- **价值放大**：单点场景的成功如果能为上级主管部门提供数据支持或监管便利，其支付意愿和推广速度将呈指数级增长[^1]。

## 场景评估公式
在原有 [[Concept_Pain_Index.md]] 基础上引入乘数：
$$场景总价值 = 痛苦指数 \times 网络价值乘数$$

## 战略应用
- **高优先级筛选**：优先开发具备高网络价值乘数的场景，如 DRG/DIP 自动控费、危急值闭环管理等。
- **价值翻译**：将技术指标翻译为对外部网络的贡献值，以此说服医院决策层[^1]。
