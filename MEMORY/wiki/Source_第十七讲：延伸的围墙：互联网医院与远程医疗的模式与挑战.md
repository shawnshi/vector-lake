# 第十七讲：延伸的围墙：互联网医院与远程医疗的模式与挑战

## 核心摘要
本讲探讨了互联网医院作为实体医院“数字化延伸”的角色。其核心目标不再是单纯的流量获取，而是服务存量、赋能基层与管理全程。同时分析了远程诊断中心在解决医疗资源分布不均中的关键作用。

## 关键观点
- **打破围墙的协同进化**: 互联网医院应定位为医院能力的延伸，核心价值在于“服务存量、赋能基层、管理全程”。
- **远程诊断中心的核心驱动**: 解决基层“看得见、看不懂”的痛点，通过影像、病理、心电的集中诊断实现资源下沉 [支持:: [[Entity_远程诊断中心.md]]]。
- **战略错位的挑战**: 许多互联网医院面临“建而不用”的窘境，根源在于过度追求外部新客而忽视了对内部存量患者的全程服务。

- **[[Entity_远程诊断中心.md]]**: 远程医疗的核心执行节点。
- **[[Concept_在线复诊.md]]**: 互联网医院的基本盘与合规边界。

## 总结
互联网医院与远程医疗正在重新定义医院的物理边界。通过数字化的“延伸围墙”，医疗服务得以实现从院内向院外、从高层向基层的结构化迁徙。
