# Concept: 闭环管理 (Closed-Loop Management)

## 核心定义
闭环管理是指在医疗流程中，针对某一特定任务（如给药、输血、危急值处理），从发起（医嘱）、执行、核对、反馈到最终评估形成的完整、可追溯的信息环路。其本质是通过系统强制保障流程安全，减少人为差错[^1]。

## 典型场景
- **医嘱闭环**: 确保“正确的病人、正确的药物、正确的剂量、正确的途径、正确的时间”。
- **危急值闭环**: LIS 发现危急值 -> 自动触发提醒 -> 医生确认并处理 -> 系统记录处理结果[^2]。
- **医技深度融合**: PACS/LIS 不再是孤立科室软件，而是通过闭环深度嵌入临床流，实现影像/检验申请到结果反馈的全生命周期监控[^2]。

## 战略意义
- **安全防线**: 消灭流程中的“断裂点”，防止因报告积压或沟通断裂导致的医疗事故。
- **数据确权**: 每个环节的签字（电子签名）确立了责任边界，解决 [[Concept_Responsibility_Black_Hole.md]]。
- **评审硬指标**: 是互联互通测评和电子病历等级评审的核心考核点[属于:: [[Concept_EMR_Rating.md]]]。

## 来源备注
## 核心案例：抗生素输注旅程
CIS 系统中的典型闭环不仅是流程连接，更是责任确权。以抗生素输注为例[^2]：
1. **医嘱开立**：医生下达医嘱，系统自动触发合理用药检查（CDSS）。
2. **药房审核**：药师通过平台审核医嘱，系统记录时间节点。
3. **配药/发药**：药房准时配药，产生物理流转条码。
4. **护理校验**：护士在床旁通过 PDA 扫描患者腕带与药袋（三查七对）。
5. **输注记录**：自动记录输注开始与结束时间。
6. **效果评估**：临床医师根据体温、血象等后续数据回馈闭环。
