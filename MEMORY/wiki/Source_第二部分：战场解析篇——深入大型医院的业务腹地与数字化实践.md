# Source: 第二部分：战场解析篇——深入大型医院的业务腹地与数字化实践

## 核心内容摘要
本章节作为“战场解析篇”，深入剖析了大型医院的业务构成与数字化转型的实战边界。核心逻辑在于：数字化不再是边缘的辅助工具，而是直接切入医疗核心业务的“手术刀”。

### 关键洞察
1. **业务腹地定义**：医院的数字化必须覆盖从门诊、急诊到住院的全流程，特别是临床决策支持（CDSS）与运营管理的深度融合。
2. **架构即战略**：系统的先进程度定义了医院数字化的天花板。
3. **翻译者角色**：医疗 IT 厂商的使命是将复杂的政策与临床需求翻译为可落地的数字化方案 [支持:: [[Entity_Winning_Health.md]]]。

## 提取的主张与证据
- **主张**：数字化系统必须具备“业务感知”能力，而非单纯的流程记录[^1]。
- **证据**：通过对大型三甲医院业务流的解构，证明了只有深度嵌入临床路径的系统才能产生真实价值。
