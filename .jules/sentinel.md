## 2024-05-04 - Fix DOM XSS in Topology Graph Info Panel
**Vulnerability:** DOM-based Cross-Site Scripting (XSS) in `templates/topology.html` where node names and sources dynamically loaded from Markdown data were interpolated directly into the DOM using `innerHTML`. This included injecting variables directly into `onclick` attributes, risking arbitrary script execution via crafted knowledge graph nodes.
**Learning:** In a Markdown-driven application without strict sanitization of source entity names, dynamically rendering those attributes natively via `.innerHTML` directly opens an avenue for XSS. Additionally, injecting variables directly into inline JavaScript event handlers (like `onclick="selectNode('${id}')"`) violates proper HTML entity isolation and frequently leads to syntactic edge-cases if attributes contain quotes.
**Prevention:** Always use safe DOM traversal manipulation methods such as `document.createElement()` paired with `.textContent` or `data-*` attributes for dynamic properties. For legacy `innerHTML` usages, implement strict `.escapeHTML()` sanitization prior to rendering strings.
