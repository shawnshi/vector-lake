## 2024-04-29 - [DOM XSS in templates/topology.html via innerHTML]
**Vulnerability:** The topology UI generates HTML using `innerHTML` with unsanitized node properties like `node.name`, `targetId`, `group`, and `source`. For example, `links.innerHTML += \`<li><a onclick="selectNode('${targetId}')">[${group}] ${name}</a></li>\`;` introduces a serious DOM-based Cross-Site Scripting (XSS) vulnerability. An attacker can create a markdown wiki file with a malicious name or group, which will execute arbitrary JavaScript when the topology visualizer is rendered and interacted with. Furthermore, directly interpolating dynamic data into inline event handlers like `onclick` can cause syntax errors and XSS.
**Learning:** Directly appending dynamic content to `innerHTML` instead of using `textContent`/`innerText` or DOM API creation exposes the app to DOM XSS. Node properties parsed from files in a knowledge graph are implicitly trusted but should be treated as untrusted user input since these files could be modified or created by anyone contributing to the knowledge base.
**Prevention:** Always use safe DOM APIs like `.textContent` or `document.createElement()` and `appendChild` when inserting dynamic data. Do not use inline event handlers with interpolated values; attach event listeners or use data-* attributes. Create an escape HTML utility function for templates if `innerHTML` must be used for layout reasons.
