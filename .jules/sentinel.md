## 2025-02-24 - DOM XSS in Topology HTML Template
**Vulnerability:** Found multiple instances where dynamic node data (`node.group`, `node.id`, `node.name`, link IDs) were directly interpolated into HTML strings and appended to the DOM via `innerHTML` without prior sanitization. Most critically, link IDs were embedded directly into `onclick="selectNode('...')"`.
**Learning:** These practices pose DOM-based XSS risks, especially since these node strings originate from markdown files. Escaping dynamic values before inserting into `innerHTML` is crucial. Also, directly embedding variables into inline JS event handlers (`onclick`) can cause syntax errors even if escaped, so `data-*` attributes should be used.
**Prevention:** Apply `escapeHTML` to all dynamic values before adding them to `innerHTML`. For interactive elements like links with custom arguments, inject the argument via a `data-*` attribute and retrieve it in the inline handler (e.g. `onclick="selectNode(this.getAttribute('data-id'))"`) or use `.addEventListener()`.
