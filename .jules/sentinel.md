## 2025-02-27 - Fix DOM XSS in topology graph renderer
**Vulnerability:** Found multiple DOM-based Cross-Site Scripting (XSS) vulnerabilities in `templates/topology.html` where unsafe `.innerHTML` assignments and template literal interpolations were used with unescaped node data (like `node.name`, `node.id`, `node.group`, and other `node` attributes) within functions like `selectNode()` and `renderGraph()`.
**Learning:** This existed because node metadata originating from user-created/managed markdown files was directly written into the DOM without sanitization, trusting the source material. Also, JavaScript template interpolations within `onclick` event handlers led to direct JS execution capabilities.
**Prevention:** Always use `document.createElement()` and assign text to the element via `.textContent` for dynamic values instead of `.innerHTML`. For rendering HTML structures dynamically using strings (like `meta` properties), first apply an `escapeHTML()` function (converting input to textNode and retrieving innerHTML) before interpolation. For dynamic event listeners, utilize HTML `data-*` attributes (`this.getAttribute('data-id')`) instead of directly injecting parameters into stringified handlers.
